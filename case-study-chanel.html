Context

  • Built during an internship in CHANEL Europe's Advanced Analytics and Data Science team.
  • Project served finance stakeholders who needed stronger trust in SAP-driven accounting data.
  • Because the underlying production data and workflows are confidential, shared artifacts focus on the method.

Problem

  • Global thresholds were not sufficient because many anomalies were only abnormal within a specific categorical branch.
  • Some risky transactions had normal-looking amounts but suspicious combinations of categorical attributes.

Constraints

  • Data mixed numeric amounts with high-cardinality hierarchical categorical fields from business systems.
  • Outputs had to remain interpretable and easy to triage in finance review workflows.

Approach

  • Built two complementary detectors: one for amount anomalies conditioned on transaction context, and one for anomalous categorical combinations.
  • Used hierarchy-aware categorical representations so parent-child category structure remained informative.
  • Generated ranked anomaly candidates with clear reason signals to support analyst investigation.

Results

  • Delivered a reusable algorithmic workflow for mixed-type outlier detection in hierarchical accounting data.
  • Published a technical note summarizing the method: Combination Outlier Detection.

My Role

Led the method design and implementation for a hierarchy-aware anomaly detection workflow during the CHANEL Europe data science internship.

Key Decisions

  • Separated amount anomalies from categorical-combination anomalies so finance users could inspect different risk signals.
  • Used hierarchy-aware categorical representations to preserve parent-child business structure.

Tradeoffs

  • Balanced model sophistication with interpretability because the output needed to fit finance review workflows.
  • Kept public artifacts method-focused to respect confidentiality while still showing technical depth.

What I Learned

  • Business context can be the difference between noise and a useful anomaly signal.
  • Trustworthy data quality tools need explanations that match how analysts investigate records.
